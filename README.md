# What I Learned About GitHub

- **Repositories:** I explored how to create and manage GitHub repositories, which act as the central place for code, history, and project context.
- **Issues:** I learned how to open and close issues, which help track tasks, questions, and bugs in an organized way.
- **Remote Connections:** I learned how to connect a local computer to a remote GitHub repository so changes can be synced between both.
- **Commits and Pushes:** I learned how to commit changes locally and push updates to GitHub.
- **Branches:** I explored how to create branches so I can work on new ideas or fixes without affecting the main branch.
- **Pull Requests:** I learned how to open pull requests and merge them into the main branch after review.
- **Forks:** I explored how forks work and how they let me make a copy of a repository for independent work.
- **GitHub Projects:** I learned how to create and use GitHub Projects to organize work and track progress.
- **GitHub Actions:** I explored how GitHub Actions can help automate workflows and repetitive tasks.
